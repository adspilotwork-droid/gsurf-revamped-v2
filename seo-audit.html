01 · The three channels

What SEO, AEO and GEO actually mean.

Most "SEO" audits cover only one channel — traditional search engine ranking. Since 2024, two more visibility channels have emerged, and both matter for a local B2B fibre operator. Here's the clean definition we've audited against.

SEO

Search Engine Optimization

Getting your pages to rank in the classic blue-link list on Google, Bing and DuckDuckGo. Driven by titles, meta descriptions, headings, backlinks and technical health.

AEO

Answer Engine Optimization

Getting your site cited by AI chatbots — ChatGPT, Perplexity, Claude, Gemini, Copilot — when users ask questions. Driven by FAQ schema, structured data, and conversational content patterns.

GEO

Generative Engine Optimization

Getting surfaced in Google's AI Overviews — the AI-generated summaries that now appear above the classic search results for 13%+ of all queries. Driven by LocalBusiness schema, GeoCoordinates, and entity consistency.

For an ISP serving Whitefield businesses, the three channels work differently. Traditional SEO drives "fibre broadband Whitefield" searches. AEO drives "which ISP should I pick for a hospital in Bangalore?" conversational queries in ChatGPT. GEO drives the AI Overview that appears above the map pack for "internet providers near me." The revamp addresses all three explicitly.

02 · Methodology

How the previous site was audited.

The audit was conducted directly on the live site at https://www.gsurf.in and on the linked broadband-plans page at gaxiom.in/broadband.php. The review covered:

HTML source inspection

Full read of the rendered HTML — meta tags, heading hierarchy, alt text, structured data, canonical URLs.

Content & information architecture

Assessed how content is organised, whether pages target specific queries, and if the user journey supports conversion.

Structured data validation

Checked for JSON-LD schema — Organization, LocalBusiness, Service, FAQPage, BreadcrumbList — and where absent, documented the gap.

Technical SEO health

robots.txt, sitemap.xml, canonical tags, HTTPS, heading structure, mobile responsiveness, page speed indicators.

Local SEO signals

Pincode references, city-specific content, GeoCoordinates, areaServed, neighbourhood-level coverage pages.

Competitive landscape

Reviewed how GSurf currently appears on third-party directories like Sulekha, Justdial, and Quora — where many prospects first discover local ISPs.

04 · Findings

What we found on the previous site.

Each issue below was directly observed on the live gsurf.in site. The "Previous state" column describes what was found; the "Fixed in revamp" column describes what the new build does differently.

Finding 01
Single-page architecture — entire site is one HTML document
SEO AEO GEO
Severity: High
Previous state

The existing gsurf.in site is a single HTML page with anchor-link navigation (#home, #about, #plans, #contact). Google, Bing and AI crawlers see one URL with mixed-intent content. There is no way to rank separately for "enterprise broadband Whitefield" vs "home broadband Bangalore" vs "ISP near ITPL" because only one URL exists.

Fixed in revamp

Rebuilt as a 16-page static site with dedicated URLs for each intent: enterprise plans, each of six ICP segments, four priority neighbourhoods, coverage, residential, about, contact. Each URL has its own title, meta description, canonical, and structured data. Google can now rank 16 different pages against 16 different queries.

Finding 02
Zero structured data — no JSON-LD schema anywhere on the site
AEO GEO
Severity: High
Previous state

The HTML source contains no JSON-LD, no microdata, no RDFa. Search engines and AI crawlers have no machine-readable business identity, no service definitions, no coverage area, no contact information in structured form. In 2026, LocalBusiness schema is the single strongest signal for inclusion in Google AI Overviews for "near me" queries.

Fixed in revamp

Full schema stack deployed: Organization, LocalBusiness (with InternetServiceProvider subtype), WebSite, Service, OfferCatalog (with all 9 plan tiers), FAQPage (on 11 pages), BreadcrumbList, Place + GeoCoordinates (on neighbourhood pages). Total: 49 JSON-LD blocks across the site, all validated.

Finding 03
Generic page title — no city, no service, brand inconsistency
SEO
Severity: High
Previous state

The live site's <title> tag reads "Gsurf - High-Speed Internet Service Provider | Enterprise Broadband". Missing: city (Whitefield, Bangalore), specific service type (fibre, FTTH), geographic targeting. Also, brand name is rendered as "Gsurf" in the title tag but "GSurf" throughout body copy — inconsistent capitalization signals weak brand authority to crawlers.

Fixed in revamp

Each of the 16 pages has a unique, keyword-rich title that includes the service (fibre broadband), the city (Whitefield / Bangalore), and the specific page intent. Brand rendered consistently as "GSurf" on every page. Example: "Enterprise Fibre for Software Companies — GSurf Whitefield".

Finding 04
No meta description — Google generates its own snippet from body text
SEO AEO
Severity: High
Previous state

No <meta name="description"> tag in the HTML source. Google auto-generates the search result snippet from the first body text it finds, which is "Ultra Fast Internet ! Better Service !!" — a tagline with no keywords and no conversion hook.

Fixed in revamp

Unique, hand-written meta descriptions on all 16 pages, each 150–160 characters, each including the page's primary keyword, the city, and the primary value proposition. AI chatbots pull these directly as entity descriptions — they are critical for AEO.

Finding 05
Multiple H1 tags on the same page — heading hierarchy broken
SEO
Severity: High
Previous state

The live site's HTML source contains multiple <h1> tags ("Ultra Fast Internet !", "We Provide Unlimited Network For You", "Enterprise Broadband Plans"). Google expects one H1 per page as the primary topic declaration. Multiple H1s dilute topical relevance.

Fixed in revamp

Every page in the revamp has exactly one H1 that states the page's primary topic. All other headings are H2, H3, H4 in correct hierarchy. Verified programmatically across all 16 pages.

Finding 06
No Open Graph or Twitter Card tags
SEO AEO
Severity: Medium
Previous state

No og:* or twitter:* meta tags. When a prospect shares the URL on WhatsApp, LinkedIn, or Twitter, the preview is blank — no image, no title, no description. AI crawlers also consume OG tags as a secondary description source.

Fixed in revamp

Full Open Graph and Twitter Card tags on every page, each with page-specific title and description. Link previews render correctly on WhatsApp, LinkedIn, Slack, X/Twitter, iMessage and Telegram.

Finding 07
No canonical URLs
SEO
Severity: Medium
Previous state

No <link rel="canonical"> tags. Without canonicals, Google doesn't know whether gsurf.in, www.gsurf.in, http://, or https:// variants are the preferred URL. Any inbound link authority gets diluted across variants.

Fixed in revamp

Every page has an explicit canonical tag pointing to its absolute URL on https://www.gsurf.in. Combined with a recommended 301 redirect rule on the host, non-www and http variants consolidate link authority into one canonical set of URLs.

Finding 08
No sitemap.xml or robots.txt
SEO AEO
Severity: Medium
Previous state

Neither file appears to exist at standard paths. Google discovers the site by following links, but without a sitemap there's no signal about which pages are important or how frequently they update. Without robots.txt, you can't explicitly opt into AI crawler access (important for AEO).

Fixed in revamp

Full sitemap.xml generated with all 16 URLs, priorities (home 1.0, enterprise 0.95, neighbourhoods 0.8–0.9, ICPs 0.75–0.85), change frequencies, and lastmod dates. robots.txt explicitly allows GPTBot, ChatGPT-User, PerplexityBot, ClaudeBot, Google-Extended and anthropic-ai — opt-in AI crawler access is now the AEO baseline.

Finding 09
No FAQ content — zero AEO-optimized question/answer pairs
AEO GEO
Severity: Medium
Previous state

No FAQ section, no Q&A formatting, no FAQPage schema. AI chatbots like ChatGPT and Perplexity preferentially cite pages that answer specific questions ("Is GSurf available in HSR Layout?", "What is the install time in Whitefield?"). Without FAQ content, there's nothing for them to extract and cite.

Fixed in revamp

FAQ sections added to 11 pages (enterprise.html, all 6 ICP pages, all 4 neighbourhood pages). Each FAQ is both rendered visibly on the page (for users) AND encoded as FAQPage JSON-LD (for AI crawlers). 49 question/answer pairs total across the site.

Finding 10
No neighbourhood or pincode-specific pages
SEO GEO
Severity: High
Previous state

The live site mentions "Whitefield" generically but has no pages targeting specific pincodes. Competitor ISPs (ACT, Airtel Xtream, Jio) have dedicated pages like "Broadband in Whitefield 560066" that win hyper-local searches. Without them, GSurf loses the highest-intent, highest-conversion traffic: people searching with their exact pincode.

Fixed in revamp

Four dedicated neighbourhood pages built: Whitefield (560066), Brookefield & Marathahalli (560037), ITPL & Hoodi (560048), Mahadevapura & KR Puram (560016). Each has its own H1, meta description, GeoCoordinates in schema, ICP mapping, install timing, recommended plan, and FAQ — all referencing the specific pincode and local landmarks.

Finding 11
Logo alt text reads "Netrun Fibernet" — wrong brand entity
SEO
Severity: Medium
Previous state

The live site's logo image has alt="Netrun Fibernet" — likely a legacy artifact from a predecessor brand. Google reads alt text as a brand signal. Mixed brand identity (GSurf / Netrun Fibernet / Gaxiom) weakens the entity graph that AI systems use to identify your business.

Fixed in revamp

Brand identity normalized across all 16 pages: "GSurf" is the public brand, "Gaxiom Networking Solutions Pvt Ltd" is the legal entity (only in Organization schema and footer disclaimers). No "Netrun Fibernet" references anywhere. Wordmark renders as a text logo with G-tile icon — no image alt text issues.

Finding 12
No language or region meta tags
GEO
Severity: Low
Previous state

No lang="en-IN" on the HTML tag, no geo.region, no geo.position. Google doesn't have explicit signals that this is a Bangalore-India business — it infers from content, which is slower and weaker.

Fixed in revamp

Explicit language + region meta tags on every page: lang="en" on HTML, content-language="en-IN", geo.region="IN-KA", geo.placename="Whitefield, Bangalore", plus geo.position and ICBM coordinates.

06 · Structured data

Every schema type deployed, and what it does.

Schema markup is the "official script" AI systems read when they need to verify facts about your business. Here's what's deployed, where, and which channel each schema type serves.

Schema type Deployed on What it does
Organization All 16 pages Declares the parent brand entity, legal name, contact points, founding date. Anchors the entity graph.
LocalBusiness (InternetServiceProvider) All 16 pages The #1 signal for Google AI Overviews on local queries. Includes NAP, geo-coordinates, areaServed (7 pincodes), opening hours.
WebSite All 16 pages Declares the site as a web property with a defined publisher and language.
BreadcrumbList 15 pages (all except homepage) Declares the page's position in the site hierarchy. Produces breadcrumb trails in Google search results.
Service 6 ICP pages + 4 neighbourhood pages Declares each ICP offering and each geographic offering as a distinct service with provider, areaServed, and OfferCatalog.
OfferCatalog + 9 Offer blocks enterprise.html Each of the 9 fibre plans rendered as a structured Offer with monthly + annual pricing and currency. AI can quote prices directly.
FAQPage enterprise.html + 6 ICP + 4 neighbourhood = 11 pages Each FAQ section encoded so AI chatbots can extract and cite individual Q&A pairs. Matched to visible on-page content.
Place + GeoCoordinates 4 neighbourhood pages Each pincode page declares its lat/lng, making the page eligible for inclusion in radius-based AI queries like "fibre near my office".

09 · Still required

What you still need to do.

The on-page SEO is complete. These are the tasks that happen after deployment and cannot be done by the website code alone.

01 · Claim & verify Google Business Profile

Single biggest local SEO lever. Claim at business.google.com, verify via postcard, add categories, photos, service areas. Link to https://www.gsurf.in.

02 · Set up Google Search Console

Add the domain, verify via DNS TXT record, submit sitemap at /sitemap.xml. Check weekly for indexing coverage and fix any errors.

03 · Configure domain redirects

Force all traffic to https://www.gsurf.in. Redirect http → https, non-www → www at the host level (Cloudflare / Hostinger / GitHub Pages custom domain settings).

04 · Run Rich Results Test

Test 3 pages at search.google.com/test/rich-results — homepage, enterprise.html, whitefield.html. Confirm Organization, LocalBusiness, FAQPage, OfferCatalog validate cleanly.

05 · Replace placeholder content with real data

Customer counts per neighbourhood, specific landmark references, latency claims — these are plausible placeholders in the copy. Confirm with the client and replace before announce.

06 · Build a review pipeline on GBP

AI Overviews and Google Maps both weigh review count and freshness heavily. Ask happy enterprise customers to leave Google reviews monthly. Reply to every review within 48 hours.
